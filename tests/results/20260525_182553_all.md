## Fixes applied during this run

Three issues in the test runner were fixed before a clean pass:

1. **PowerShell `$ErrorActionPreference` + `2>&1` interaction** — hurl and k6 write progress to stderr. With `$ErrorActionPreference = "Stop"`, `2>&1` wraps stderr lines as `NativeCommandError` objects which become terminating errors. Fix: toggle `$ErrorActionPreference` to `"Continue"` around native command invocations that capture stderr.

2. **Hurl `file,` path resolution** — hurl resolves `file,<path>` body references relative to its own resolution root, not the caller's CWD. Fixture paths like `tests/fixtures/wake-payload.json` failed when the hurl file lived in `tests/hurl/`. Fix: pass `--file-root $RepoRoot` to hurl invocations.

3. **Hurl JSONPath `count` on single-match filters** — `jsonpath "$.events[?(@.type=='agent_start')]" count >= 1` errors when the filter matches exactly one element (returns object, not list). Fix: use `exists` predicate instead of `count >= 1`.
