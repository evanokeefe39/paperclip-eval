# M0.1 Failure Fixes — Subagent Execution Plan

Source: `tasks/evaluations/m0.1-postmortem.md`

## Key findings from exploration

- `setup.sh` discover_agents() at lines 302-323 globs all subdirs of src/agents/ — no compose service filter
- `bridge.mjs` invocation lock (lines 81-88) uses `while(invocationLock) { await invocationLock }` — queues instead of rejecting
- `triage-workflow.ts` has pre-hooks only (Pi `tool_call` event, line 214). Can intercept tool input BEFORE execution. No post-hooks.
- `artifact-service` content_hash column has INDEX but no UNIQUE constraint. No dedup check in handleWrite.
- `artifacts/index.ts` passes `run_id: params.run_id` from tool params only (line 70). Does NOT read PAPERCLIP_RUN_ID from env.
- `artifacts/client.ts` write() accepts run_id as optional parameter, passes to HTTP body.

## Wave 1 — 5 parallel subagents

All file sets are disjoint. Each subagent writes a failing test + the fix.

### W1-A: F1 — Ghost agent registration guard
- **Files:** `src/agents/setup.sh`, `tests/e2e/e2e-ghost-agents.sh`
- **Depends on:** none
- **Changes:** In discover_agents() (line 302), parse compose service names and skip agent dirs that don't match. Test: parse compose, compare to agent dirs, assert coder/publisher/qa would NOT be registered.

### W1-B: F2 — Bridge 503 on lock contention
- **Files:** `src/agents/bridge.mjs`, `tests/bridge/test-invocation-lock.mjs`
- **Depends on:** none
- **Changes:** Replace await-queue lock with immediate 503 response when lock held. Test: send two concurrent /invoke requests, verify second gets 503 with Retry-After header.

### W1-C: F3 — CEO self-assignment guard
- **Files:** `src/agents/extensions/triage-workflow.ts`, `tests/triage/test-self-assign-guard.mjs`
- **Depends on:** none
- **Changes:** In tool_call hook (line 214), add check: if tool is paperclip_create_issue and input.assigneeAgentId === PAPERCLIP_AGENT_ID, block with reason "CEO cannot self-assign issues". Test: simulate tool_call event with self-assignment, verify it's blocked.

### W1-D: F4 — Artifact content_hash dedup
- **Files:** `src/artifact-service/routes.ts`, `src/artifact-service/metastore.ts`, `tests/artifact-service/test-dedup.sh`
- **Depends on:** none
- **Changes:** Add findByContentHash() in metastore. In handleWrite, check for existing hash before insert — return existing artifact if found. Test: POST same content twice, verify second returns existing ID.

### W1-E: F5 — Auto-populate run_id from env
- **Files:** `src/agents/extensions/artifacts/index.ts`, `tests/artifact-service/test-run-id.sh`
- **Depends on:** none
- **Changes:** In write_artifact tool (line 70), default run_id to process.env.PAPERCLIP_RUN_ID when not explicitly passed. Test: set PAPERCLIP_RUN_ID env, call write_artifact without explicit run_id, verify S3 key contains the run_id.
