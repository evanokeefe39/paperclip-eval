# M1.0: Agent Recruitment & Continuous Improvement

Self-improving workforce. When a capability gap is identified, the workforce can research and propose how to fill it — either by recruiting a new agent or improving an existing one.

## Recruitment

Capability gap detected (manually or by agents during work) triggers the workforce to research what's needed: extensions, skills, tools, data sources, system prompts, model selection. Output is a fully specified agent proposal ready for onboarding.

## Improvement

Periodic or triggered review of existing agents. Workforce searches the web for better skills, extensions, source material, techniques. Proposes upgrades with evidence.

## RLHF / Fine-tuning

Introduce feedback loops and model fine-tuning to improve agent outcomes over time. Leverage run traces, human corrections, and quality scores as training signal.

## Success Criteria

- [ ] Workforce can autonomously research and propose a new agent spec given a capability gap description
- [ ] Proposed agent specs are actionable — contain concrete extensions, skills, model, prompt direction
- [ ] Workforce can audit an existing agent and propose evidence-backed improvements
- [ ] Improvement proposals reference real tools/skills/extensions found on the web, not hallucinated ones
- [ ] At least one RLHF or fine-tuning loop demonstrated end-to-end
- [ ] Recruited or improved agent demonstrably performs better than before on a real task

## What This Tests

Whether the workforce can improve itself — closing the loop from "we have a gap" to "gap filled" with minimal human intervention.
