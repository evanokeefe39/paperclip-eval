### 2.2 Implement `transcribe_audio` tool

File: `src/agents/extensions/web-scrape.ts`

New tool registered when `GROQ_API_KEY` env var is present:

```typescript
transcribe_audio({
  video_url: string,      // URL to video file (MP4)
  language?: string,       // default "en"
})
```

Logic:
1. Download video to temp file (curl/fetch)
2. Extract audio via ffmpeg: `ffmpeg -i input.mp4 -vn -acodec libmp3lame -q:a 4 output.mp3`
3. POST audio to Groq `/v1/audio/transcriptions` with model `whisper-large-v3-turbo`
4. Return transcript text
5. Clean up temp files

Rate limiting:
- Read `x-ratelimit-remaining-requests` from Groq response headers
- If remaining < 5, log warning
- If 429, wait for `x-ratelimit-reset-requests` duration and retry once

## Phase 3: NIM Video Analysis Integration

Effort: ~2 hrs. New tool in web-scrape.ts extension.

### 3.1 Implement `analyze_video` tool

File: `src/agents/extensions/web-scrape.ts`

New tool registered when `NVIDIA_NIM_API_KEY` env var is present:

```typescript
analyze_video({
  video_url: string,       // publicly accessible URL to MP4
  focus?: string,          // optional focus area (e.g. "product placement", "branding")
})
```

Logic:
1. Validate URL is accessible (HEAD request, check content-type)
2. POST to NIM `/v1/chat/completions` with:
   - Model: `nvidia/nemotron-3-nano-omni-30b-a3b-reasoning`
   - Content: `video_url` type + structured JSON prompt
   - max_tokens: 2048, temperature: 0.1
3. Parse JSON response
4. Return structured analysis object

Prompt template (from spike):
```
Return ONLY a JSON object analyzing this video:
{
  "summary": "2-3 sentence summary",
  "topics": ["topic1", "topic2"],
  "on_screen_text": ["text1", "text2"],
  "tone": "one word",
  "speakers": [{"name": "...", "said": "brief quote"}],
  "transcript_summary": "key points from speech",
  "visual_details": "products, branding, demos, graphics not in subtitles"
}
No markdown, no explanation, just the JSON.
```

### 3.2 Client-side rate limiter

NIM exposes no rate limit headers. Implement a simple token bucket:

```typescript
class RateLimiter {
  constructor(rpm: number) // default 40
  async acquire(): Promise<void> // blocks until a slot is available
}
```

Shared instance across all NIM calls in the extension.

## Phase 4: Orchestration — `enrich_video` Composite Tool

Effort: ~1 hr. Convenience tool that chains the pipeline.

### 4.1 Implement `enrich_video` tool

File: `src/agents/extensions/web-scrape.ts`

```typescript
enrich_video({
  video_url: string,       // publicly accessible URL
  platform: "tiktok" | "youtube" | "instagram",
  subtitle_text?: string,  // pre-existing subtitle text (from Apify)
})
```

Logic:
1. If `platform === "instagram"` and no `subtitle_text`: call `transcribe_audio`
2. Call `analyze_video` with the URL
3. Merge subtitle text + visual analysis into unified content object
4. Return combined result

This is a convenience wrapper — agents can also call the individual tools directly.

### 4.2 Update agent prompts

Data and Researcher agent prompts should mention `enrich_video` as available
for extracting rich content from scraped social media videos.
