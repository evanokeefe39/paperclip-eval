Source: `tasks/evaluations/m0.1-postmortem.md`

- [ ] Publish structured workproduct findings to artifact service (not just aggregate markdown docs). Individual findings with ADMIRALTY grades are queryable, composable, and auditable. Currently they exist only on ephemeral workspace volumes — lost on `docker compose down -v`.

## P1 — Correctness

- [ ] Fix CEO self-assigning research tasks. CEO executed EVA-62 and EVA-66 despite prompt forbidding direct research. Add triage-workflow hook or Paperclip-level constraint blocking CEO from assigning research/data issues to itself.
- [ ] Add content_hash deduplication to artifact service writes. Agents published same document 2-5 times under different artifact_types. Reject duplicate content_hash unless force flag set.
- [ ] Fix stale context contamination. CEO created 3 false escalation issues (EVA-57, 58, 69) from old crash-loop comments. Add wipe step to setup.sh fresh-start flow, or add context freshness checks.

## P2 — Performance

- [ ] Investigate MiniMax timeout pattern. 5 timeouts in 30 minutes on long tool-use chains. Consider Pi-level keepalive, streaming heartbeat, or DeepSeek fallback for complex tool flows.
- [ ] Skip heartbeat processing when active run is in progress. Heartbeat wakes during active work waste 1-2 turns processing "check inbox" prompts.
- [ ] Fix duplicate write task creation. CEO created 3 overlapping write tasks (EVA-56, 67, 68). Improve CEO decomposition prompt or add pre-creation dedup check.

## P3 — Data quality

- [ ] Fix S3 key structure. Most artifacts use `no-run` and `default` placeholders. Propagate run_id and company_id from bridge env through to artifact client.
- [ ] Enforce artifact_type controlled vocabulary. Current types are free-text. Define enum (research, brief, report, dataset, finding) and validate on write.
- [ ] Populate encouraged workproduct fields (topic_tags, entities, related_findings). Add prompt engineering or post-processing to fill these.
